# 全体公開にする方法
Vercel ダッシュボードで プロジェクト vercel-test のデプロイ保護を弱める／オフにします。
- 開く: vercel-test → Settings（左下歯車） → Deployment Protection
- Vercel Authentication を設定
- 本番を外部に見せたい: 一番上の「Enabled for」の青いトグルを OFF にする


# ビルドしたページを表示する方法
- Vercel で vercel-test を開く
- Settings（左下歯車） → General
- Build and Deployment →Framework Preset を Next.js に変更する
- Save
- Deployments タブに戻り、最新の Production を Redeploy（できれば一度 キャッシュなし）


# ベーシック認証
AIにベーシック認証入れろと指示
Vercel → プロジェクト vercel-test → Settings → Environment Variables

次を追加（Production にチェック）
## 1. 最初の変数（ユーザー名）
- 右上の 「Add Environment Variable」（黒ボタン）をクリック
- Key に BASIC_AUTH_USER と入力
- Value に表示名用のユーザー名（例: demo）を入力
- Environments で Production にチェック（Preview / Development にもかけたいならその分もチェック）
- Save（または Add／Save と書かれたボタン）

## 2. 2 本目（パスワード）
もう一度 「Add Environment Variable」
- Key: BASIC_AUTH_PASSWORD
- Value: 推測されにくい文字列（テスト用でも最低限の長さはあるとよい）
- Environments は上と同じく Production（必要なら他も）
- Save
- Deployments で最新を Redeploy（変数はデプロイ時に読み込まれます）
